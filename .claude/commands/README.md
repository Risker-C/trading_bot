# 自定义命令使用说明

## 可用命令

### /feature-dev - 功能开发标准流程

**用途**: 标准化功能开发流程，强制执行完整的开发、测试、文档、部署流程

**触发方式**:
- 自动触发：当用户提出功能开发需求时自动使用
- 手动触发：输入 `/feature-dev` 命令

**执行流程**:
1. 需求分析与规划
2. 开发实施
3. 文档编写（必须）
4. 测试用例编写（必须）
5. 测试执行（本次+全部）
6. 服务验证（必须重启）
7. 代码提交和推送
8. 总结报告

**质量标准**:
- ✅ 代码质量：遵循现有风格，有错误处理和注释
- ✅ 文档质量：结构完整，内容清晰，有示例
- ✅ 测试质量：覆盖核心功能，所有测试通过
- ✅ 部署质量：服务正常运行，无错误日志

**文件规范**:
- 文档位置：`docs/{功能名称}.md`
- 测试位置：`scripts/test_{功能名称}.py`
- 测试集成：自动添加到 `test_all.py`

**使用示例**:
```
用户: 新增一个定时备份功能
系统: [自动使用 feature-dev 流程]
      阶段1: 需求分析...
      阶段2: 开发实施...
      ...
```

## 命令验证

要验证命令是否正确配置，可以：

1. 检查命令文件是否存在：
   ```bash
   ls -lh .claude/commands/feature-dev.md
   ```

2. 查看命令内容：
   ```bash
   cat .claude/commands/feature-dev.md
   ```

3. 测试命令（输入）：
   ```
   /feature-dev
   ```
